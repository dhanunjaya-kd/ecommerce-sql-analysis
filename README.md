E-Commerce Analytics using SQL
Project Overview

This project analyzes an e-commerce dataset using PostgreSQL to uncover sales trends, customer behavior, product performance, and payment insights.

The objective is to transform raw transactional data into meaningful business insights using SQL queries and data analysis techniques.

Tools Used
-PostgreSQL
-SQL
-CSV Dataset
-GitHub

Dataset Information
The project uses the Brazilian Olist E-Commerce Dataset containing:
-Customers
-Orders
-Order Items
-Products
-Payments
Total records analyzed: 100,000+ transactions

Business Questions Answered
-What is the total revenue generated?
-Which product categories generate the highest revenue?
-What are the most used payment methods?
-Which states contribute the most customers?
-What is the average order value?
-What are the monthly sales trends?
-Which customers generate the highest revenue?
-Which product categories drive the most orders?

Key Insights
-Revenue Performance
-Total Revenue: ₹16.01 Million
-Total Orders: 99K
-Average Order Value: ₹160.99 

Customer Analysis
-Total Customers: 96K+
-Customer base is concentrated in a few high-performing states.
-Revenue per customer is approximately ₹166.59.

Product Analysis
-Beauty & Health category generated the highest revenue.
-Watches & Gifts and Bed/Bath products were among the strongest performing categories.
-Product revenue is highly concentrated among top categories.

Payment Analysis
-Credit Card is the dominant payment method.
-Boleto is the second most used payment option.
-Voucher and Debit Card contribute a relatively small share of revenue.

Freight Analysis
-Total Freight Cost: ₹2.25 Million.
-Certain product categories contribute disproportionately to shipping costs.
